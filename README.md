# 硬幣金額辨識 (Coin Amount Recognition)

這是一個使用 Python 與 OpenCV 實作的硬幣金額辨識程式。程式會讀取包含硬幣的影像，透過影像處理技術偵測出硬幣的位置與大小，並根據硬幣的半徑大小來判斷面額，最後計算出圖片中的總金額。

## 功能特色

- **影像預處理**：包含影像縮放 (500x500)、灰階轉換、中值濾波 (Median Blur) 等處理。
- **邊緣偵測**：使用 Laplacian 運算子進行邊緣特徵強化。
- **圓形偵測**：透過霍夫圓變換 (Hough Circle Transform) 找出影像中硬幣的位置與半徑。
- **面額辨識**：根據偵測到的圓形半徑大小，判斷硬幣面額（支援新台幣 50元、10元、5元、1元 硬幣）。
- **結果輸出**：在影像上圈出硬幣位置、顯示總金額，並在終端機輸出各面額硬幣的數量與總計。
- **過程記錄**：程式執行過程中會將各階段的影像處理結果（灰階圖、模糊圖、邊緣偵測圖）以及最終結果分別儲存為圖片檔。

## 環境需求

- Python 3.x
- OpenCV (`opencv-python`)
- NumPy (`numpy`)

你可以使用以下指令安裝所需的套件：
```bash
pip install opencv-python numpy
```

## 使用方法

1. 將你要辨識的硬幣圖片準備好，命名為 `coin.jpg`，並放置於與 `main.py` 相同的目錄下。
2. 執行主程式：
```bash
python main.py
```
3. 程式執行完畢後，終端機會印出各面額硬幣的數量以及總金額。
4. 目錄下會產生以下處理過程及結果的圖片：
   - `gray.jpg`：灰階影像
   - `median.jpg`：中值濾波處理後的影像
   - `Laplacian.jpg`：Laplacian 邊緣偵測後的影像
   - `output.jpg`：標記了硬幣位置 (紅色圓圈) 及總金額的最終結果影像

## 硬幣判斷邏輯

程式透過硬幣在 500x500 影像中呈現的半徑大小 (r) 來判斷面額。目前的判斷標準如下：
- `r > 55`：50 元
- `49 < r < 55`：10 元
- `42 < r < 49`：5 元
- `30 < r < 42`：1 元

> **注意**：硬幣面額的辨識準確度高度依賴拍攝距離、相機角度與光線。若辨識結果不佳，或是使用了不同拍攝環境的圖片，建議手動調整 `main.py` 中的半徑判斷條件 (`coinPrice` 函式)，或是微調霍夫圓變換 (`cv2.HoughCircles`) 的相關參數。
